## 三種掛載：一張圖分清楚

容器要碰「可寫層以外」的儲存，只有三條路：

三條路的一句話定位與選擇口訣：

- **volume**：Docker 幫你管的儲存空間，住在 `/var/lib/docker/volumes/`。生命週期獨立於容器、可備份可搬家、跨平台行為一致——**生產環境的資料一律用它**。
- **bind mount**：把主機的某個目錄直接接進容器，主機改、容器立刻看到。**開發時共享程式碼、掛設定檔用它**；缺點是綁死主機路徑、權限議題多。
- **tmpfs**：掛一塊記憶體當目錄，容器一停內容蒸發、永不落磁碟。**敏感暫存資料用它**。
- 選擇口訣：**資料要活得比容器久用 volume；要跟主機即時同步用 bind；連磁碟都不准碰用 tmpfs**。
